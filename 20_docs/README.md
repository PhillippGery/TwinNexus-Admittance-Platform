# 20_docs

*Project technical documentation for architecture, networking, and coordinate-frame definitions.*

## Contents

- `architecture/` - diagrams and subsystem interaction documents
- `network/` - UR control box IP plans, port references, and workstation connectivity notes
- `tf_frames/` - TF tree definitions, frame naming references, and calibration relationships

## Conventions

- Diagrams should use stable filenames with a short topic-oriented slug.
- Network documents should separate lab-local values from reusable topology guidance.
- TF/frame documentation must use ROS frame naming consistently and note parent-child assumptions explicitly.
- Documentation here should describe system behavior and layout, not serve as an implementation dump.
