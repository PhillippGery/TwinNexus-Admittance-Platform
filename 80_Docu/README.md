# 80_Docu

*Research-facing documentation, papers, reports, and analysis outputs for the project.*

## Contents

- `papers/` - references, annotated readings, and manuscript-related material
- `reports/` - project status summaries, milestone notes, and performance writeups
- `analysis/` - experiment summaries and derived findings intended for human review

## Conventions

- Keep filenames human-readable and sortable, ideally with a date or revision prefix where useful.
- Reports should state experiment context, system revision, and dataset references when applicable.
- Analysis outputs stored here should be interpretable on their own without requiring notebook execution context.
- Source data belongs in `30_data/`; this folder is for curated documentation artifacts.
